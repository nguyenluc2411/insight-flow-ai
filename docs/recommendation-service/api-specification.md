REST API Specification
Recommendation Service
Insight Flow AI – Engagement Layer
1. API Overview
1.1 API Responsibilities

The recommendation-service REST API provides operational and analytical access to recommendation data generated by the recommendation engine.

The API is responsible for:

retrieving generated recommendations
filtering recommendations
viewing recommendation details
exposing recommendation analytics
supporting operational monitoring
enabling manual recommendation triggering (administrative use)

The API does not:

execute inventory actions directly
modify inventory quantities
perform AI forecasting
1.2 Interaction with Dashboard Service

The dashboard-service acts as the primary API consumer.

Dashboard Responsibilities

The dashboard-service consumes recommendation APIs to:

display recommendation summaries
render inventory risk dashboards
visualize recommendation analytics
provide operational management interfaces
Communication Pattern
Frontend
    ↓
dashboard-service (BFF)
    ↓
recommendation-service API
1.3 Interaction with Frontend

Frontend applications should access recommendation data through the dashboard-service BFF layer rather than calling recommendation-service directly.

Benefits
Benefit	Description
Aggregation	Dashboard combines multiple services
Security	Centralized authentication
Reduced Coupling	Frontend isolated from internal services
Optimized Payloads	UI-specific responses
2. Endpoint Design
Base URL
/api/v1/recommendations
2.1 Get Recommendations
Endpoint
GET /api/v1/recommendations
Purpose

Retrieve paginated recommendations with filtering support.

2.2 Get Recommendation Details
Endpoint
GET /api/v1/recommendations/{recommendationId}
Purpose

Retrieve detailed information about a specific recommendation.

2.3 Get Recommendation Analytics
Endpoint
GET /api/v1/recommendations/analytics
Purpose

Retrieve aggregated recommendation statistics.

2.4 Get Recommendation History
Endpoint
GET /api/v1/recommendations/{recommendationId}/history
Purpose

Retrieve lifecycle history of a recommendation.

2.5 Manual Recommendation Trigger
Endpoint
POST /api/v1/recommendations/trigger
Purpose

Manually trigger recommendation generation for operational or testing purposes.

3. Request DTOs
3.1 RecommendationFilterRequest
Purpose

Supports recommendation filtering and pagination.

DTO Structure
{
  "recommendationType": "CLEARANCE",
  "riskLevel": "HIGH",
  "status": "GENERATED",
  "productId": "uuid",
  "startDate": "2026-05-01",
  "endDate": "2026-05-31",
  "page": 0,
  "size": 20,
  "sortBy": "generatedAt",
  "sortDirection": "DESC"
}
Validation Rules
Field	Validation
recommendationType	Enum validation
riskLevel	Enum validation
status	Enum validation
page	Min = 0
size	Min = 1, Max = 100
sortDirection	ASC or DESC
3.2 ManualRecommendationTriggerRequest
DTO Structure
{
  "productId": "uuid",
  "warehouseId": "uuid",
  "triggerReason": "Manual operational review"
}
Validation Rules
Field	Validation
productId	NotNull
triggerReason	NotBlank, max 500 chars

Query Parameters
Parameter	Description

Validation failures return standardized error responses.

Example

Status	Meaning

Status	Meaning

Status	Meaning

Conclusion

The recommendation-service REST API is designed following enterprise-grade Spring Boot REST architecture principles.

The API provides:

scalable recommendation retrieval
flexible filtering and pagination
secure JWT-based access
clean DTO-based responses
standardized error handling
maintainable endpoint structure

The design supports both operational dashboards and future frontend integrations while maintaining strong microservice boundaries and clean backend architecture principles.
